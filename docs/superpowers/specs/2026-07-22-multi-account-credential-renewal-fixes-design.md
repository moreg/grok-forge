# 多账号凭据续期修复设计

## 目标

修复多账号凭据续期改造中的安全、迁移可靠性和历史任务连接问题，不改变账号数据模型或正常账号切换流程。

## 范围

1. 不再向 WebView 暴露可读取或写入原始钥匙串令牌的 Tauri 命令。旧钥匙串的读取、导入和删除只由后端迁移命令完成，前端只接收脱敏状态。
2. 旧钥匙串迁移必须幂等：已有目标认证文件时先检查其可读性和有效状态；仅确认目标已成功迁移后删除旧钥匙串项；已删除的旧项视为成功重试。
3. 自动连接不得把未归属任务传入账号隔离层。当前任务没有账号归属时，自动连接保持等待；用户确认绑定后，使用已绑定账号重新发起连接。
4. 同步调整单元测试，使其覆盖上述回归路径，并修正 Windows 换行差异造成的 Rust 测试失败。

## 设计

### 凭据边界

删除 `get_secure_token`、`store_secure_token` 及前端兼容包装器。账号注册表继续只保存元数据和旧钥匙串项标识。`migrate_keyring_credential` 是唯一读取旧项的命令，返回续期能力、认证状态、过期时间和账号标识，不返回令牌。

### 迁移重试

当目标 `auth.json` 已存在时，后端先调用凭据检查。文件不可读、格式损坏或没有可用认证条目时返回错误且保留旧钥匙串项。文件有效时，删除旧项；钥匙串项已不存在表示先前迁移已完成，仍返回目标检查结果。首次迁移仍保持“原子写入成功后才删除源项”的顺序。

### 历史任务连接

连接函数要求任务账号与当前账号均存在且相等。未归属任务不会自动尝试 ACP 连接，避免消耗一次性自动连接机会。用户选择未归属任务并确认绑定后，任务会清空 ACP 会话并恢复自动连接；新建任务和已归属任务保持现有行为。

### 验证

新增最小回归测试：桥接层不再导出明文令牌方法、迁移重试不会在目标损坏时删除源项、未归属任务不会调用 ACP 启动、绑定后可以启动。执行桌面端测试、生产构建、Tauri `cargo check` 与库测试。
